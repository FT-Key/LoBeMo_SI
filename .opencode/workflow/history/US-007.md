# US-007: Gestión de tareas

## Card Info

- **Trello Card:** https://trello.com/c/xkZCHFAf
- **Estado:** En Progreso
- **Tamaño:** M

## Descripción

Como Empleado técnico, quiero crear y gestionar mis tareas dentro de un proyecto para organizar el trabajo y registrar mi avance.

## Acceptance Criteria

- AC-01: Las tareas tienen: título, descripción, estado (PENDIENTE, EN_PROGRESO, COMPLETADA, CANCELADA) y prioridad.
- AC-02: Un empleado solo puede crear/modificar tareas en proyectos donde está asignado.
- AC-03: El CISO y Gerente General pueden ver y modificar todas las tareas del proyecto.
- AC-04: Al completar una tarea CRÍTICA, se notifica al CISO (RN-15d).
- AC-05: El sistema valida RN-06 al marcar un proyecto como ENTREGADO.

## Plan

### Archivos a crear (2)
1. `src/app/api/tareas/route.ts` — API: GET listar + POST crear tareas
2. `src/app/api/tareas/[id]/route.ts` — API: GET single + PATCH actualizar + DELETE eliminar

### Archivos a modificar (2)
3. `src/app/proyectos/[id]/page.tsx` — Pasar información de asignaciones del usuario actual
4. `src/app/proyectos/[id]/proyecto-detalle.tsx` — Gestión completa de tareas (CRUD inline)

### Funcionalidades
- Crear tarea con título, descripción, prioridad, fecha límite
- Validar que el empleado esté asignado al proyecto (RF-30)
- CISO y Gerente General pueden ver/modificar todas las tareas (RF-31)
- Actualizar estado/prioridad/campos de tarea
- Eliminar tarea (solo CISO o Gerente General)
- Notificar al CISO al completar tarea CRÍTICA (RN-15d)
- AuditLog en cada operación
