# US-013: Gestión de capacitaciones

## Card Info
- **Trello Card**: [vMgEsgSd](https://trello.com/c/vMgEsgSd)
- **Tamaño**: M
- **Prioridad**: Should Have

## Description
Como Capacitador en Ciberseguridad, quiero crear programas de capacitación, registrar sesiones y evaluar asistentes para ofrecer servicios de formación a clientes.

## Acceptance Criteria
- AC-01: Un programa de capacitación incluye: temario, duración, modalidad (presencial/remota), materiales.
- AC-02: Se pueden registrar asistentes (empleados del cliente) y su evaluación.
- AC-03: Se genera un certificado digital al completar la capacitación.

### Modelo de datos

#### Capacitacion
- id (UUID)
- proyecto_id (FK → Proyecto, nullable)
- titulo (string)
- temario (text)
- duracion_horas (int)
- modalidad (enum: PRESENCIAL, REMOTA)
- fecha_inicio (date)
- fecha_fin (date)
- estado (enum: PLANIFICADA, EN_CURSO, COMPLETADA, CANCELADA)
- materiales (text, nullable - URLs o descripción)
- created_at, updated_at

#### AsistenteCapacitacion
- id (UUID)
- capacitacion_id (FK → Capacitacion)
- nombre_asistente (string)
- email_asistente (string)
- organizacion (string, nullable)
- evaluacion (int, nullable - 1-10)
- completado (boolean, default false)
- created_at, updated_at

#### CertificadoCapacitacion
- id (UUID)
- asistente_id (FK → AsistenteCapacitacion, unique)
- codigo_certificado (string, unique)
- fecha_emision (date)
- created_at

### Decisiones técnicas clave
- Se implementaron las reglas de negocio RN-CAP-01 a RN-CAP-04 en las API routes.
- El código de certificado sigue el formato `CERT-{capacitacion_id}-{asistente_id}-{YYYYMMDD}` (RN-CAP-04).
- La generación del certificado solo es posible cuando `completado=true` (RN-CAP-03).
- Se utilizó arquitectura de server components para las páginas y client components para los formularios e interacciones.
- Las validaciones de roles se implementaron a nivel de API routes con verificación de sesión.

### Reglas de negocio
- RN-CAP-01: Solo CAPACITADOR y GERENTE_GENERAL pueden crear/modificar capacitaciones.
- RN-CAP-02: Solo CAPACITADOR puede registrar/evaluar asistentes.
- RN-CAP-03: Un certificado solo se genera cuando el asistente completó la capacitación (completado=true).
- RN-CAP-04: El código de certificado es único (formato: CERT-{capacitacion_id}-{asistente_id}-{YYYYMMDD}).
