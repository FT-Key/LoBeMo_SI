# US-020: Panel de Administración (Configuración del Sistema)

## Card Info
- **Tamaño:** S
- **RF-64:** El Gerente General debe poder configurar parámetros del sistema (límite de proyectos activos por empleado, días de aviso de vencimiento, etc.).

## Plan

### Archivos creados:
- `prisma/schema.prisma` — modelo `Configuracion` (clave-valor)
- `prisma/migrations/20260704042000_add_configuracion/` — migración
- `src/app/api/configuracion/route.ts` — API GET/PATCH
- `src/app/admin/page.tsx` — página servidor
- `src/app/admin/admin-view.tsx` — componente cliente (formulario)

### Archivos modificados:
- `src/app/api/asignaciones/route.ts` — usar config en lugar de hardcoded 3
- `src/app/servicios/page.tsx` — nav link a /admin (solo GG)
- `src/app/empleados/page.tsx` — nav link a /admin
- `src/app/dashboard/page.tsx` — nav link a /admin (solo GG)

### Parámetros configurables:
- `MAX_PROYECTOS_ACTIVOS_POR_EMPLEADO` (default: 3)
- `DIAS_AVISO_VENCIMIENTO_PROPUESTA` (default: 3)
- `DIAS_AVISO_HITO` (default: 3)

## Implementación

- Se creó el modelo `Configuracion` con clave única y valor string.
- GET `/api/configuracion` lista todas las configuraciones (solo GG).
- PATCH `/api/configuracion` actualiza/crea configuraciones con validación.
- Se integró `MAX_PROYECTOS_ACTIVOS_POR_EMPLEADO` en la validación RN-08 del endpoint de asignaciones.
- UI con formulario numérico, validación, feedback de éxito/error.
