## Description
**Como** Empleado técnico, **quiero** adjuntar documentos (informes, código, configuraciones) a proyectos y tareas **para** centralizar la documentación de cada servicio.

## Acceptance Criteria
- AC-01: Se pueden subir archivos PDF, imágenes, documentos de Office y archivos de texto.
- AC-02: Los documentos se tipifican (INFORME_AUDITORIA, REPORTE_PENTESTING, etc.).
- AC-03: Solo el equipo del proyecto puede ver sus documentos; Gerencia y CISO ven todos.

## Plan

### Files to create:
1. `src/app/api/documentos/route.ts` — GET list + POST create document
2. `src/app/api/documentos/[id]/route.ts` — GET single + DELETE document

### Files to modify:
3. `src/app/proyectos/[id]/proyecto-detalle.tsx` — Add "Documentos" section with upload/list

### Design decisions:
- File storage: base64 data URL stored in `Documento.url` (MVP approach, swappable to Vercel Blob)
- Auth: employees assigned to project can upload/view; GERENTE_GENERAL and CISO can view all
- Validation: file size limited client-side, type classification required
- AuditLog: CREATE and DELETE operations logged
