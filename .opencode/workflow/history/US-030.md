# US-030: Validaciones de formularios con Zod compartido

## Card Info
- **Trello Card**: Por crear
- **Description**: Implementar validaciones de formularios en frontend y backend usando Zod + react-hook-form, con schemas compartidos entre cliente y servidor.
- **Acceptance Criteria**:
  - AC-01: Todos los formularios del frontend deben usar react-hook-form + Zod para validación en tiempo real con mensajes de error por campo.
  - AC-02: Todas las rutas API (POST/PATCH) deben validar el body usando los mismos schemas Zod del frontend.
  - AC-03: Los schemas deben vivir en `src/shared/validation/` y reutilizarse entre cliente y servidor.
  - AC-04: Enums (roles, sectores, prioridades, etc.) deben estar centralizados en los schemas compartidos.
  - AC-05: La página /register debe redirigir a /login si ya existe un superadmin.
  - AC-06: CtaSection debe ocultar el link a /register si ya existe un superadmin (consistente con HeroSection).
