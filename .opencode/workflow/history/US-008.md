# US-008: Gestión de hitos

## Card Info

- **Trello Card:** https://trello.com/c/z770Gnm5
- **Estado:** En Progreso
- **Tamaño:** S

## Descripción

Como CISO, quiero definir hitos dentro de un proyecto para controlar las fechas clave y el progreso del equipo.

## Acceptance Criteria

- AC-01: Los hitos tienen: nombre, descripción, fecha prevista y fecha real de cumplimiento.
- AC-02: El sistema notifica 3 días antes de la fecha prevista de un hito.
- AC-03: El Gerente General y el CISO pueden crear/modificar hitos.

## Plan

### Archivos a crear (2)
1. `src/app/api/hitos/route.ts` — API: GET listar + POST crear hitos
2. `src/app/api/hitos/[id]/route.ts` — API: GET single + PATCH actualizar + DELETE eliminar

### Archivos a modificar (1)
3. `src/app/proyectos/[id]/proyecto-detalle.tsx` — Gestión completa de hitos (CRUD inline)

### Funcionalidades
- Crear hito con nombre, descripción, fecha prevista
- Solo CISO y Gerente General pueden crear/modificar hitos (RF-36)
- Actualizar campos, marcar como completado con fecha real automática
- Eliminar hito (solo CISO o Gerente General)
- Indicador visual de hitos próximos (3 días o menos)
- AuditLog en cada operación
