# US-016: Gestión de tickets de soporte técnico

## Card Info
- **Trello Card**: https://trello.com/c/hTdJkq5h
- **Estado**: In Progress
- **Tamaño**: M (3-4 días)

## Descripción
**Como** Soporte Técnico, **quiero** gestionar tickets de asistencia de clientes vinculados a proyectos **para** dar seguimiento a incidentes y solicitudes.

## Acceptance Criteria
- AC-01: El Soporte Técnico puede crear tickets con título, descripción, prioridad y categoría.
- AC-02: Los tickets pueden vincularse a un proyecto existente.
- AC-03: Los tickets tienen estado: ABIERTO, EN_PROCESO, RESUELTO, CERRADO.
- AC-04: Se puede cambiar el estado del ticket con validaciones de transición.
- AC-05: El sistema lista tickets con filtros por estado, prioridad y proyecto.
- AC-06: Solo los roles autorizados (SOPORTE_TECNICO, GERENTE_GENERAL, CISO) pueden gestionar tickets.
- AC-07: Cada operación CRUD se registra en AuditLog (RN-09).

## Implementación

### Prisma Schema
Agregar modelo `TicketSoporte` con:
- id, titulo, descripcion, prioridad (BAJA, MEDIA, ALTA, CRITICA)
- estado (ABIERTO, EN_PROCESO, RESUELTO, CERRADO)
- categoria (INCIDENTE, CONSULTA, SOLICITUD)
- clienteId?, proyectoId?, creadorId, asignadoAId?

### API Routes
- `GET /api/soporte` — Lista paginada con filtros (estado, prioridad, proyecto)
- `POST /api/soporte` — Crear ticket (requiere SOPORTE_TECNICO, GERENTE_GENERAL, CISO)
- `GET /api/soporte/[id]` — Obtener detalle
- `PATCH /api/soporte/[id]` — Actualizar campos o estado
- `DELETE /api/soporte/[id]` — Eliminar (solo si está ABIERTO)

### Páginas
- Lista con tabla, filtros, paginación
- Formulario de creación
- Detalle con cambios de estado
