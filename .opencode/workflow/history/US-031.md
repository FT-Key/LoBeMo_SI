# US-031: Fix login cookie name mismatch on HTTPS (deploy)

## Card Info
- **Trello**: N/A (Trello no conectado)
- **Estado**: ✅ Done

## Problema
En local (`npm run start`) el login funciona correctamente, pero en el deploy (Vercel, HTTPS):
- Se ingresan credenciales correctas
- El server action devuelve `{ success: true }`
- Se muestra "Redirigiendo..."
- Se navega a `/dashboard` pero inmediatamente redirige a `/login`
- El formulario se limpia, sin errores en consola

## Causa Raíz
El server action en `src/app/(auth)/login/actions.ts` setea la cookie de sesión con:
- Nombre: `authjs.session-token`
- `secure: false`

En producción (HTTPS), NextAuth v5 detecta HTTPS y `auth()` busca la cookie como `__Secure-authjs.session-token` (con prefijo `__Secure-` y `secure: true`). Al no encontrarla, `auth()` devuelve null y redirige a `/login`.

En local (HTTP), NextAuth usa `authjs.session-token`, que coincide con lo que setea el server action → funciona.

## Solución
Modificar `src/app/(auth)/login/actions.ts` para determinar dinámicamente:
- El nombre de cookie correcto según el protocolo (`__Secure-authjs.session-token` para HTTPS, `authjs.session-token` para HTTP)
- El flag `secure` acorde

Se usa `process.env.AUTH_URL` (cuando comienza con `https://`) y `process.env.NODE_ENV` como fallback para determinar si se requiere cookie segura.
