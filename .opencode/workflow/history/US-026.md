# US-026: Fix login session persistence — cookie name mismatch

## Card Info
- **Trello**: [US-026](https://trello.com/c/qjZ5Se9u)
- **Estado**: ✅ Done

## Problema
El login validaba credenciales pero no persistia la sesion. Al hacer submit:
- Chrome mostraba notificacion de contraseña (el form se enviaba correctamente)
- Los campos se limpiaban
- Redirigia a /dashboard pero inmediatamente volvia a /login

## Causa Raíz
1. **Cookie name mismatch**: El middleware (`src/middleware.ts`) buscaba la cookie de sesion como `next-auth.session-token` (nombre de NextAuth v4), pero NextAuth v5 la establece como `authjs.session-token`
2. **signIn() no persiste cookies en Server Actions**: NextAuth v5 `signIn()` llama a `cookies().set()` internamente, pero en el contexto de Server Actions de Next.js 16 no funciona correctamente

## Solucion
- Reemplazar `signIn()` con creacion manual del JWT usando `encode()` de `@auth/core/jwt` y `cookies().set()` directo
- Corregir nombre de cookie en middleware de `next-auth.session-token` a `authjs.session-token`
